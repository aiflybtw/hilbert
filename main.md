# Цель и задачи

**Цель:** разработка матрицы грейдов для специалистов DevOps с описанием
уровней, основными требованиями, обоснованными ориентирами по заработной
плате, составленная на основании анализа рынка труда.

**Основные задачи:**

1.  Организация сбора и хранения данных по вакансиям.

2.  Фильтрация и предобработка данных.

3.  Экстракция релевантной информации, в частности требуемых навыков и
    опыта, обязанностей из текстов вакансий.

4.  Проведение кластеризации навыков.

5.  Построение модели для оценки факторов, влияющих на грейд
    специалиста.

6.  Построение модели для получения обоснованной интервальной оценки
    заработных плат специалистов разных грейдов.

7.  Использование полученных результатов при построении матрицы грейдов.

## Источники данных

Данные по вакансиям собираются с трех источников:

-   **hh.ru** --- крупнейший агрегатор вакансий в РФ.

# Предобработка

## Удаление дубликатов

Дубликатами признавались вакансии, где одинаковая должность, вилка
заработной платы, грейд, также одинаковая компания-публикатор.

## Нормализация оплаты труда

Перед построением модели заработной платы были выполнены следующие шаги:

1.  Идентификация пропущенных маркеров валют при помощи применения
    регулярных выражений к описаниям. Для тех вакансий, где после этого
    по-прежнему не удалось идентифицировать валюту описания направлялись
    в LLM с просьбой идентифицировать валюту по расположению офиса
    компании или другим маркерам. Вывод модели был верифицирован
    вручную. После этого этапа вакансий, где не удалось установить
    валюту компенсации не осталось.

2.  Перевод всех компенсаций в рубли был осуществлен при помощи запросов
    к API Банка России для конвертации по актуальным обменным курсам.

3.  Идентификация не месячных компенсаций проводилась при помощи
    эвристик. Если компенсация меньше 10 тыс. руб. --- оплата почасовая,
    если оплата больше 2 млн. руб. --- компенсация годовая.

4.  Приведение почасовых и годовых оплат к месячному формату (формат
    5/2 - 40 рабочих часов в неделю). Годовые компенсации были разделены
    на 12, почасовые --- умножены на 160.

# Признаки для кластеров навыков

## Hard Skills

Вместо бинарных признаков используется непрерывная мера ---
*разнообразие эмбеддингов* навыков внутри кластера.

**Ключевой принцип:** чем разнообразнее инструменты вакансии в рамках
одного кластера, тем больше множитель. Многие инструменты одного
кластера являются взаимозаменяемыми альтернативами, поэтому прямой
подсчет вхождений некорректен.

Для пары (вакансия $i$, кластер $j$) вычисляется *эффективное
количество*:
$$\text{EffectiveCount}_{ij} = k_{ij}\cdot(1 - \text{MPCS}_{ij}),
  \label{eq:effcount}$$ где $k_{ij}$ --- количество навыков из
кластера $j$ в вакансии $i$, $\text{MPCS}_{ij}$ --- Mean Pairwise Cosine
Similarity эмбеддингов этих навыков.

Для снижения чувствительности к вакансиям с большим числом навыков
применяется логарифмическое сжатие:
$$\text{HardFeature}_{ij} = \ln(1 + \text{EffectiveCount}_{ij}).
  \label{eq:hard_feat}$$

## Soft Skills

Аналогичный подход применяется к мягким навыкам. Поскольку первоначально
кластеризация была выполнена без генерации эмбеддингов для софт-скиллов
внутри каждого из 6 кластеров требовалось отдельно сгенерировать
описания, а затем эмбеддинги. На основании полученных эмбеддингов для
каждого кластера рассчитывался аналогичный показатель
([\[eq:soft_feat\]](#eq:soft_feat){reference-type="ref"
reference="eq:soft_feat"}).
$$\text{SoftFeature}_{ij} = \ln(1 + \text{EffectiveCount}_{ij}).
  \label{eq:soft_feat}$$

## Responsibilities

Учитывая формат извлеченных через LLM обязанностей критически необходимо
было снизить кол-во уникальных пар действие + объект. Кластеризация была
реализована следующими этапами:

1.  Приведение всех встречающихся форм слова действий к единому формату
    --- отглагольному существительному.

2.  Разделение составных действий, например изменение \<\<Поддержка и
    Равитие\>\> в \<\<Поддержка / Развитие\>\>.

3.  Анализ и обработка часто встречающихся паттернов с разным
    написанием, например \<\<Настрока CI/CD пайплайнов; Поддержка
    пайплайнов CI/CD; Разработка CI/CD пайплайнов\>\>.

4.  Получение эмбеддингов для нормализованных пар действие + объект,

# Анализ требований к грейдам

## Спецификация модели

Основными подзадачами в данном разделе являются, во-первых,
идентификация навыков и. обязанностей, встречающихся в вакансиях на том
или ином грейде, а во-вторых, дискриминативный анализ навыков, то есть
определение факторов, отделяющих один грейд от другого.

Для решения первой подзадачи построение модели не требуется: достаточно
провести частотный анализ и определить, какие кластеры навыков и
требований чаще всего встречаются внутри каждого грейда. Такой анализ
показывает типичный профиль вакансии, но не позволяет отделить
специфические признаки грейда от навыков, распространенных на всех
уровнях.

Вторая подзадача требует построения модели классификации. В качестве
объясняющих переменных используются признаки кластеров, полученные
ранее, кроме кластеров обязанностей ввиду большой корреляции с навыками:
значения $\text{HardFeature}_{ij}$ для кластеров hard skills по
формуле [\[eq:hard_feat\]](#eq:hard_feat){reference-type="ref"
reference="eq:hard_feat"} и значения $\text{SoftFeature}_{ij}$ для
кластеров soft skills по
формуле [\[eq:soft_feat\]](#eq:soft_feat){reference-type="ref"
reference="eq:soft_feat"}. Для каждой вакансии $i$ формируется вектор
признаков
$$\mathbf{x}_i = \bigl(1_{i},\text{HardFeature}_{i1},\ldots,\text{HardFeature}_{iJ_h},
  \text{SoftFeature}_{i1},\ldots,\text{SoftFeature}_{iJ_s}\bigr),
  \label{eq:grade_feature_vector}$$ где $J_h$ и $J_s$ --- число
кластеров hard skills и soft skills соответственно.

Целевая переменная $y_i$ принимает одно из пяти значений:
$$y_i \in \mathcal{G} = \{\text{Intern},\text{Junior},\text{Middle},\text{Senior},\text{Lead}\}.$$
Для оценки связи между набором навыковых кластеров и грейдом
используется многоклассовая логистическая регрессия:
$$\Pr(y_i = g \mid \mathbf{x}_i) =
  \frac{\exp(\mathbf{x}_i^{\top}\bm{\beta}_g)}
  {\sum\limits_{h \in \mathcal{G}} \exp(\mathbf{x}_i^{\top}\bm{\beta}_h)},
  \quad g \in \mathcal{G},
  \label{eq:grade_multinomial_logit}$$ где $\mathcal{G}$ --- множество
грейдов, а $\bm{\beta}_g$ --- вектор коэффициентов при признаках
кластеров.

Коэффициенты модели интерпретируются как дискриминативные признаки
грейдов. Положительное значение коэффициента при кластере для грейда $g$
означает, что наличие и разнообразие навыков из данного кластера
повышает логарифм шансов отнесения вакансии к этому грейду при прочих
равных. Отрицательное значение же указывает, что кластер скорее отделяет
вакансию от данного грейда.

## Результаты дискриминативного моделирования

В этой модели нас в первую очередь интересует не точность классификации
грейдов, а то, какие навыковые кластеры сильнее всего отличают один
грейд от другого. Поэтому стандартные метрики качества классификации,
например Accuracy, Precision, Recall или F1-мера, здесь не являются
основным результатом: они показывают, насколько хорошо модель
предсказывает тот или иной класс, но не отражают содержательные различия

отдельно: они показывают, действительно ли существует дискриминативная
структура, позволяющая отличать классы, на которой в свою очередь

# Моделирование зарплатной вилки

## Спецификация модели

Обозначим фактическую зарплату специалиста как $\text{Salary}_i^*$,
нижнюю границу вакансии как $L_i$, а верхнюю --- как $U_i$. Тогда
наблюдается интервал $[L_i,\,U_i]$ со строгим включением
$L_i < \text{Salary}_i^* < U_i$. Это соответствует предпосылкам
*интервальной регрессии*. Поскольку распределение заработных плат
приближено к логнормальному, принята следующая спецификация:
$$\ln\text{Salary}_{i}^{*}
    = \mathbf{x}_{i}^{\top}\bm{\beta} + \varepsilon_{i},
  \label{eq:base}$$ где $$\ln \text{Salary}_{i}^{*} \in
  \bigl[\ln L_{i},\;\ln U_{i}\bigr].$$

Дополнительным преимуществом данной модели является совместная обработка
всех трех типов вакансий:

-   известны обе границы: $[L_i,\,U_i]$;

-   известна только верхняя граница $U_i$;

-   известна только нижняя граница $L_i$.

Ожидая гетероскедастичность ошибок (в частности, по грейду), расширяем
спецификацию: $$\begin{aligned}
  \ln\!S_{i}^{*}
      &= \mathbf{x}_{i}^{\top}\bm{\beta}
         + \sigma_{i}\,\varepsilon_{i},\quad
         \varepsilon_{i}\sim\mathcal{N}(0,1),
  \label{eq:hetero}\\
  \sigma_{i} &= \exp(\mathbf{g}_{i}^{\top}\bm{\gamma}),
  \label{eq:sigma}
\end{aligned}$$ где $\mathbf{g}_{i}$ задает дисперсию, например через
грейд специалиста. Экспоненциальная
форма [\[eq:sigma\]](#eq:sigma){reference-type="eqref"
reference="eq:sigma"} обеспечивает неотрицательность $\sigma_{i}$.

Параметры оценивались при помощи Метода Максимального Правдоподобия в
предположении нормальности логарифма заработной платы.

## Построение интервалов

### Способ 1

Исходя из задачи построения интервального прогноза по полученным оценкам
$\hat{\bm{\beta}}$, $\hat{\bm{\gamma}}$ для нового наблюдения
$\mathbf{x}_0$ c грейдом $\mathbf{g}_0)$ получаем оценку
\<\<логзарплаты\>\> $\hat\mu_0 = \mathbf{x}_{0}^{\top}\hat{\bm{\beta}}$
и стандартного отклонения
$\hat\sigma_0 = \exp(\mathbf{g}_{0}^{\top}\hat{\bm{\gamma}})$.

Тогда для самой заработной платы можно опредить квантиль уровня $p$
следующим образом:
$$Q_{p}\!\left(\text{Salary}\mid\mathbf{x}_0,\mathbf{z}_0\right)
  = \exp\!\bigl(\hat\mu_0 + z_p\,\hat\sigma_0\bigr),
  \label{eq:quantile}$$ где $z_p$ --- квантиль стандартного нормального
распределения уровня $p$. При $p=0.5$ получаем медиану заработной платы,
которая выступит прогнозной величиной - центром прогнозного интервала.

Введем параметр $\alpha$ и построим прогнозный интервал:
$$= \bigl[\exp(\hat\mu_0 + z_\alpha\hat\sigma_0),\;
           \exp(\hat\mu_0 + z_{1-\alpha}\hat\sigma_0)\bigr].
  \label{eq:interval}$$

При $\alpha=0.1$
интервал [\[eq:interval\]](#eq:interval){reference-type="eqref"
reference="eq:interval"} интерпретируется следующим образом: *для
требований $\mathbf{x}_0$ около 80% наблюдаемых зарплат лежат внутри
этого диапазона.*

### Способ 2

Альтернативным способом является получение доверительных интервалов
через бутстреп. Он помогает преодолеть основной недостаток способа 1 ---
чувствительность к качеству оценки $\sigma$. Полученная в рамках
кросс-валидации, которая призвана избежать оценку дисперсии на тех же
данных, на которых модель обучалась, оценка может быть сильно завышена
из-за огромного разброса остатков для грейдов с малым кол-вом
наблюдений. В рамках оценки через бутстреп для каждого грейда будут
сэмплированы $B = 10000$ остатков, полученны прогнозы заработных плат
$\{\tilde{\text{Salary}}_{1},...,\tilde{\text{Salary}}_{B}\}$ и итоговый
интервал будет иметь вид:
$$[\text{Lower}_{\alpha},\text{Upper}_{1-\alpha}] = [\text{Q}_{\alpha}(\tilde{\text{Salary}}),\text{Q}_{1-\alpha}(\tilde{\text{Salary}})],$$
где $\alpha$ - параметр интервала. На основании сказанного был сделан
выбор в пользу применения именно бутстрапированных интервалов.

## Оценка качества модели

Поскольку перед нами стоит задача получения интервальных прогнозов, то
будет оцениваться именно качество построенных интервалов. В частности:

1.  Интервалы должны покрывать не менее заявленного процента наблюдаемых
    зарплатных вилок по каждому грейду.

2.  При этом ширина интервалов должна быть минимальной.

линейного и образует профиль с одним резким скачком в середине
карьерного пути.

1.  Переход Intern $\rightarrow$ Junior остается относительно небольшим:
    медианная зарплата увеличивается на 10,7 тыс. руб. (+14,6%). По всей

    позиции с ограниченной самостоятельностью.

2.  Переход Junior $\rightarrow$ Middle --- единственный в карьерной
    лестнице, где наблюдается кратный рост: +108,1 тыс. руб.
    ($\times 2.28$). Именно здесь происходит качественное изменение
    роли: специалист переходит от выполнения задач под руководством к
    самостоятельной работе. Рынок труда фиксирует этот переход
    максимальной премией.

3.  Переход Middle $\rightarrow$ Senior остается значимым в абсолютном
    выражении (+115,8 тыс. руб.), однако в относительном --- уже заметно
    скромнее ($\times 1.60$). Абсолютный прирост здесь сопоставим с
    предыдущим шагом, но база сравнения существенно больше, поэтому
    относительный эффект меньше.

4.  Переход Senior $\rightarrow$ Lead также остается умеренным: рост
    медианы составляет +42,5 тыс. руб. (+13,8%). Это не свидетельствует
    об отсутствии разрыва между уровнями --- позиция Lead традиционно
    несет управленческую и архитектурную нагрузку. Скорее, это отражает
    два других эффекта: малый объем выборки (n = 20) и принципиально
    иную структуру компенсации на уровне Lead, куда все чаще включаются
    нефиксированные составляющие (бонусы, доли), не отраженные в
    зарплатной вилке вакансии.

Ширина интервала ожидаемо сокращается при переходе от 80% к 60%
покрытию. Если взвесить грейды по числу наблюдений, средняя ширина
уменьшается с 186,6 тыс. руб. до 115,3 тыс. руб., то есть примерно на
38,2%. По грейдам сокращение неоднородно: Intern --- 34,0 тыс. руб. в
обоих интервалах, Junior --- со 115,3 до 49,1 тыс. руб., Middle --- со
172,2 до 114,9 тыс. руб., Senior --- со 199,3 до 123,0 тыс. руб., Lead
--- с 248,9 до 140,2 тыс. руб. Наиболее широкий диапазон сохраняется у
Lead, что отражает высокую неоднородность требований и компенсаций
внутри старших ролей; наиболее узкий --- у Intern.\

ориентира ни рекрутеру при формировании оффера, ни руководителю при
планировании бюджета.\

тех 60%, кто не является ни существенно опнедооплачиваемым, ни редким
высококвалифицированным исключением, ширина интервала при это нв среднем
не превышает половину от заработной платы. Выбор 60%-го интервала также
подкреплен эмпирически. Как показывает
таблица [10](#tab:grade_salary_intervals_60){reference-type="ref"
reference="tab:grade_salary_intervals_60"} и рисунок
[2](#fig:salary_ranges_80_60){reference-type="ref"
reference="fig:salary_ranges_80_60"}, для грейдов Middle и Senior ---
составляющих 273 из 320 наблюдений выборки (85%) --- эмпирическое
покрытие практически совпадает с целевым уровнем: 60,4% и 59,9%
соответственно. Это означает, что для основной массы вакансий
смоделированная вилка не является ни излишне зажатой, ни избыточно
широкой. Для грейдов Intern, Junior и Lead качество интервальных оценок
ниже ввиду малого объема наблюдений (4, 23 и 20 соответственно), поэтому
для этих позиций вилки следует рассматривать как ориентировочные и
подлежащие ручной корректировке при накоплении данных.

# Ограничения исследования

1.  **Дисбаланс классов.** В выборке доминируют специалисты Middle и
    Senior. Напротив, грейды Intern, Junior и Lead представлены
    значительно слабее. Поэтому выводы для массовых грейдов являются
    более устойчивыми.

2.  **Разные источники данных.** hh.ru, career.habr.ru и Telegram-канал
    различаются по формату публикаций, полноте описания требований,
    стилю указания зарплатных вилок и аудитории работодателей. Из-за
    этого частоты навыков, обязанностей и зарплатные ориентиры могут
    частично отражать специфику площадок, а не только реальные различия
    между грейдами.

3.  **Полнота описаний вакансий.** Качество описания является важным
    конфаундером. В кратких или плохо структурированных вакансиях часть
    навыков, обязанностей, требований к опыту или признаков грейда может
    отсутствовать либо быть сформулирована неявно. В таком случае модель
    получает неполный набор признаков и может ошибочно отнести вакансию
    к более низкому или более высокому уровню компенсации.

4.  **Использование LLM.** Несколько ключевых этапов --- извлечение
    сущностей из текстов вакансий, идентификация грейда и валюты,
    нормализация навыков и кластеризация --- опираются на вывод LLM
    (`deepseek-v4-flash`). Несмотря на ручную верификацию,
    систематические ошибки модели могли сохраниться: например,
    объединение близких, но не эквивалентных навыков, пропуск редких
    технологий или неверная атрибуция обязанностей.

5.  **Временная актуальность результатов.** Данные отражают состояние
    рынка на момент сбора вакансий и выбранный горизонт наблюдения.

# Выводы и рекомендации по использованию

Стоит отметить ярко-выраженную нелинейность в росте компенсаций.
Заработная плата между специалистами Intern и Junior, а также Senior и
Lead почти не изменяется. В то же время, середина карьерного роста
Junior $\to$ Middle $\to$ Senior сопровождается достаточно активным
ростом компенсаций. Рисунок
[1](#fig:interval_width){reference-type="ref"
reference="fig:interval_width"} также показывает небольшой существующий
разрыв в уровнях заработных плат между этими уровнями, поскольку
усредненные 60%-е интервалы для этих грейдов не пересекаются. Суждения
по поводу заслуженности такого роста, а также \<\<недооцененности\>\>
или \<\<переоцененности\>\> того или иного грейда остаются за пределами
данного исследования, поскольку требуют экспертного анализа изменений в
требуемых компетициях.

Полученную матрицу грейдов можно использовать при пересмотре текущих
сотрудников, поскольку она задает \<\<структуру разговора\>\>, то есть
указывает не только на часто встречающиеся компетенции и опыт, но и на
уникальные компетенции и требования грейда, позволяющие вести более
предметный разговор с кандидатом на повышение грейда. Более того,
матрица грейдов предоставляет обоснованный ориентир для позиционирования
оффера, который может дорабатываться в зависимости от дополнительных
требований.
